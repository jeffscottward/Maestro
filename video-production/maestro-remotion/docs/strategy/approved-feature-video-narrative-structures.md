# Approved Feature Video Narrative Structures

## Purpose

This note defines the small approved structure set for the standalone Maestro feature videos. Each strategy doc should select one of these arcs instead of inventing a new story shape during scene implementation.

## Approved Structures

### `Discovery -> Activation -> Proof`

Use when the feature starts in a discovery surface and needs to show the exact moment a developer moves from browsing to committed execution, then prove the workflow is alive.

Best fit:

- `Maestro Symphony`

### `Fragmentation -> Visibility -> Synthesis`

Use when the feature collapses scattered activity into one control surface and then compresses that activity into a faster decision-making artifact.

Best fit:

- `Director's Notes`

### `Risk -> Isolation -> Handoff`

Use when the feature value is safer execution, cleaner containment, and an easier review path after the work branches away from the current context.

Best fit:

- `Run in Worktree`

## Selection Rules

- Open with a real product state that a muted viewer can understand without narration.
- Use the middle beat to show the irreversible product action, not a decorative transition.
- End on proof from the product itself: status, summary, review path, or completed outcome.
- Keep Maestro's exact in-product labels on screen even when the marketing wrapper uses different language.
